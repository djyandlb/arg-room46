📖 Base64 编码原理与识别方法

什么是 Base64？

Base64 是一种将二进制数据编码为可打印 ASCII 字符的编码方案。它使用 64 个可打印字符（A-Z、a-z、0-9、+、/）来表示二进制数据，等号（=）用作填充字符。Base64 不是加密算法——它只是编码，任何人都可以轻松解码。在 ARG 解谜中，Base64 常用于在网页源码中隐藏短文本信息。

如何识别 Base64？

Base64 编码有几个明显的特征：

  • 字符集有限： 只包含字母（大小写各26个）、数字（10个）、加号（+）、斜杠（/），末尾可能有等号（=）填充
  • 长度通常是4的倍数： 标准 Base64 的输出长度一定是4的倍数，不足时用 = 或 == 补齐
  • 隐藏在注释中： 在网页中，Base64 经常出现在 HTML 注释 <!-- ... --> 或 JavaScript 字符串中
  • 看起来像乱码但有规律： 它看起来是一串大小写混排的字母数字，偶尔以等号结尾

快速解码方法

浏览器控制台： 按 F12 打开开发者工具，在控制台输入 atob("编码内容") 即可解码。例如 atob("SGVsbG8=") 会输出 "Hello"。
在线工具： 搜索 "Base64 Decode" 可找到大量在线解码工具，适合解码较长的内容。
Node.js： 在 Node 环境中使用 Buffer.from("编码内容", "base64").toString() 进行解码。

Base64 编码原理（进阶）

Base64 的编码过程如下：将原始数据的每 3 个字节（24 位）分成 4 组，每组 6 位。6 位二进制数可以表示 0-63 的范围，对应 Base64 字符表中的 64 个字符。如果原始数据的字节数不是 3 的倍数，用等号填充。解码时反向操作：将 4 个 Base64 字符的 6 位值合并为 24 位（3 个字节），还原原始数据。
