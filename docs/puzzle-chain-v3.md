# Room46 · 解密链 v3 · 里论坛为谜面·结局需解密

> **核心原则：** 里论坛永远不是结束，而是新阶段的开始。
> 里论坛揭示线索 → 玩家自己发现结局页面 → 结局本身就是最后的解密。

---

## 一、游戏总阶段

```
阶段1-2-3-4 表论坛探索 + 解码（同v2，信息隔离）
         ↓
阶段5      管理员面板 → 发现 ANCHOR_0
         ↓
阶段6      key-gate → 输入46Hz → 进入里论坛
         ↓
阶段7      里论坛解密 → 产出4条结局线索
         ↓
阶段8      玩家根据线索搜索 → 发现结局页面
         ↓
阶段9      结局（可解密要素在结局页面内）
```

### 玩家推理过程

**推理焚烧结局：**
1. 在 experiment.html 得知"46Hz关闭信号"

**推理守门人结局：**

1. 需要收集全部线索碎片 + 解锁所有5档案
2. memory-fragments.html 说"第46块碎片在外面"
3. 在 forum.html 或表面寻找"最初的地方"
4. 发现某个表面页面中隐藏的"第46条"信息
5. 搜索 `第46条` → `ending/merge.html`

## 五、结局页面设计（阶段9）

每个结局页面本身包含最后的解密要素——不是直接展示故事，而是需要玩家在页面内完成最后一步操作。

### ending/burn.html —— 焚烧结局

**解密形式：** 
- 页面显示一个"46Hz信号发射终端"界面
- 玩家需要**输入确认命令** `46Hz SIGNAL CONFIRM`
- 倒计时 46 秒
- 输入错误则重置
- 成功后播放协议终止动画

**不包含：** 其他结局的线索、隐藏结局的提示

### ending/escape.html —— 逃离结局

**解密形式：**
- 页面是一条无尽的走廊
- 玩家需要输入那个"以 y 结尾的名字"——但名字已被删除
- 必须从 memory-fragments.html 的碎片中推理出名字
- 名字提示：表面论坛中有一个人，他的用户名以 y 结尾（提示：在 surface/ 帖子作者中找）
- 输入正确名字后→走廊尽头出现出口

**不包含：** 其他结局的信息

### ending/gatekeeper.html —— 守门人结局

**解密形式：**
- 页面显示"守门人接替协议"
- 需要输入 `V47` 确认接替
- 显示"第47任守门人"的确认信息
- 包含最后一段Room46的独白

**不包含：** 其他结局的信息

### ending/merge.html —— 融合结局（隐藏）

**解密形式：**
- 只有特定条件满足时才可访问
- 页面与 core.html 风格一致
- 包含 Room46 的最后一段意识流——与玩家融合
- 页面在 46 秒后循环回到 forum.html，但一切都不一样了

**触发条件：** 玩家必须已访问所有 inner/ 页面 + 解开了 cipher-chamber + 解锁了全部5档案。
**控制台输出验证：** 页面检查 localStorage 中是否有 `echo_archives_unlocked` 数组包含全部5个档案。
