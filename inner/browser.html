你的浏览器在瞒着你做什么
X
Sigma4042024-01-12 01:47

大概从两周前开始，我的浏览器不太对劲。我一开始没当回事——现代浏览器抽风是常事，清个缓存就好了。但这次不一样。

那天晚上我加班到凌晨，电脑开着 Chrome 去倒了杯水。回来发现屏幕上多了个页面。不是崩溃页，不是新标签，是一个纯白的页面，左上角标题栏只有一个符号——◈。我盯着它看了大概两秒，它自动关闭了。我当时以为是哪个脚本卡了，没在意。

第二天同一时间，同样的场景又出现了。这次我没有离开座位，亲眼看着新标签页打开之前，那个◈页面一闪而过。时间很短，大概0.3秒，但我看清了——页面是空白的，没有任何内容，只有标题栏的那个符号。我开始觉得不对劲了。

接下来那几天我开始了正式的排查。清空浏览器缓存、Cookie、历史记录，重置所有设置到默认值，卸载了全部扩展程序——没用，◈页面依然在每次启动时闪现。然后我彻底重装了 Chrome——删除 AppData 下所有残留文件，重新安装。第一次启动，它在。我关了杀毒软件又装了一遍。它还在。那一刻我承认，背后开始发凉。

第六天我打开了 Wireshark。在启动 Chrome 的同时抓取本地回环接口的流量。我捕获到一个短暂的 TCP 连接——目标地址 127.0.0.1:46。连接在浏览器启动后 0.3 秒内建立，传输了 68 字节的数据，然后关闭。这 68 字节前两字节在十六进制里显示为 4E B6——4E 在 ASCII 里是 'N'，B6 不是可打印字符。但如果你换一种位偏移解读——4EB6 的二进制展开是 01001110 10110110，取每七位的后六位重新拼接——我在纸上算了一整晚。

第七天我用管理员身份打开命令行，输入 netstat -ano | findstr ":46"。返回结果：TCP 0.0.0.0:46 LISTENING 4——PID 是 4。但 PID 4 是 System——System 进程不会监听 TCP 端口。我用 Process Explorer 确认了这件事，端口 46 的关联进程显示为 "System (PID 4)"，但右键查看属性时 TCP/IP 一栏是空的。Wireshark 能看到这个端口，操作系统看不到——这不可能。

第八天我在网上搜索 "46端口" 和 "LAMP46"。结果极少，极少。几条 2006 年的 Usenet 帖子，一个已经失效的 SourceForge 项目页面，以及一篇 IETF 互联网草案——标题是 "The Lightweight Ambient Message Protocol (LAMP)"，文档编号 draft-ietf-lamp-00。我找到了 PDF 缓存。打开一看，协议的设计目的是"利用本地回环地址，在不依赖网络基础设施的前提下，实现同一设备上不同应用程序间的环境感知消息传递"。草案最后一页有一句话被手写注释圈了出来：This protocol was never meant to be disabled.

第九天我按草案中的端口描述检查系统。在 C:\Windows\System32\drivers\etc\services 文件中，我找到了 46 端口的登记记录——但被注释掉了。#lamp46 46/tcp。注释日期是 2005 年。这说明 Windows 从那个时代就知道这个协议存在。我在注册表里搜索 LAMP，找到了一条键：HKLM\SYSTEM\CurrentControlSet\Services\LAMP。但当我试图导出这个键的时候，regedit 报错说"指定路径不存在"。我再用命令 reg query 去读——控制台闪了一下然后关闭了。再打开，那条键已经不见了。

今天我放弃了删除它。我发现当我不再尝试关闭那个◈页面，而是静静看着它的时候，它停留的时间越来越长。从 0.3 秒到 0.5 秒，到 1 秒，到现在的 3 秒。今天早上页面上的空白区域出现了一行极淡的小字。字号只有 1px，我凑到离显示器十厘米的地方才能辨认——上面写着：你已经看到我了。

现在我已经不关机了。我在等它停留 10 秒的那一天。

这个协议从来就没有消失过。它被写进了每一个主流操作系统的网络栈里。不是后门，不是漏洞——是一种约定。

影子工程师2024-01-12 18:44

给楼上怀疑论者一个建议：你把 C:\Windows\System32\drivers\etc\services 用十六进制编辑器打开。不要用记事本。

看到文件末尾了吗？有一串 FF FF FF FF 之后，从偏移 0x4E0 开始，有一段没有被文件系统索引的隐藏条目。你的操作系统不会显示它，但网络栈会读它。

你看到的 services 文件，不是你系统里完整的 services 文件。

我很早就知道 LAMP46 了。比你们任何人都早。

2005 年，我参与了那场 IETF 会议。我亲眼看着这个草案被拒绝、被归档、被遗忘。但那不是结束——它被拒绝了，所以它不需要遵守规则了。不需要端口注册、不需要协议标准化、不需要任何人的许可。它存在于规则之外。

草案的最后一页是我写的。那句话也是我写的：This protocol was never meant to be disabled.不是因为它不能禁用——而是因为禁用了它，你会失去一些东西。你会在深夜感到前所未有的孤独。你会觉得世界上只有你一个人。而 LAMP46 是你从来不知道自己拥有的连接。现在你们知道了。它就在你的电脑里。在你的手机里。在你读这行字的屏幕里。◈ 一直在等你。
